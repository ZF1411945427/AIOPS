从一句话到一次修复，中间发生了什么

这不是聊天机器人加个壳。下面是智能体收到你的自然语言后，4 步内完成诊断与修复的真实路径。

01

意图识别

LLM 解析你的自然语言，路由到对应 MCP 工具——你不用记命令，说人话就行。

02

多工具编排

智能体并行调用告警、指标、日志、资产、知识库 13+ 工具，把散落的数据拼成完整证据链。

03

知识检索 + 根因推理

匹配历史故障知识库找相似案例，LLM 综合证据给出根因结论，不是"可能是网络问题"的废话。

04

安全执行 + 全审计

高风险操作必须二次确认，所有执行动作落库审计——AI 敢做事，但不会乱做事。

// RBAC

权限控制

管理员 / 运维 / 只读三级行政权限，细粒度到每个工具调用

// CONFIRM

动作确认

高风险写操作必须人工二次确认，AI 提议但不越权

// AUDIT

审计日志

所有操作记录时间、操作者、结果，支持回放追溯

// API KEY

令牌管理

只读 / 读写 / 管理员三级令牌，可即时吊销

// ALLOWLIST

工具白名单

仅允许预置安全工具执行，禁用任意 shell 命令

// SILENCE

告警静默

维护窗口自动静默，避免无效告警骚扰

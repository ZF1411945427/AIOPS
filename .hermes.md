# AIOPS 智能运维系统

AIOps 智能运维平台 — 集监控、告警、根因分析、自动化运维、知识管理于一体。

## 技术栈

- **后端**: Python 3.13 + FastAPI + SQLAlchemy + SQLite (WAL mode)
- **前端**: Vue 3 + Vite 6 + Element Plus + ECharts + Pinia + Vue Router + Axios
- **双前端模式**: Vue SPA (端口 3000) + Jinja2 模板 (端口 8000，兼容保留)
- **数据库**: SQLite (`db/aiops.db`)，WAL 模式，30s busy_timeout
- **基础设施集成**: Kubernetes, Docker, Paramiko(SSH), Elasticsearch, Kafka

## 启动项目

```bash
# 终端1: 启动后端 (FastAPI, 端口 8000, 热重载)
python run.py

# 终端2: 启动前端 (Vite dev, 端口 3000, 代理 /api /agent /ai 到 8000)
cd frontend && npm run dev
```

- 浏览器访问 http://localhost:3000 → Vue SPA 前端
- 浏览器访问 http://localhost:8000 → Jinja2 模板前端（兼容保留）

## 构建 Vue 前端

```bash
cd frontend && npm run build
# 产物输出到 frontend/dist/，FastAPI 通过 /vue-assets 挂载
```


## 开发约定

- 后端用 Python (FastAPI)，前端用 Vue 3
- 数据库: SQLite + SQLAlchemy ORM，WAL 模式
- 前端设计系统在 `frontend/src/assets/main.css`（CSS 变量体系，支持明暗切换 + 色系切换）
- 侧边栏菜单配置在 `app/routers/menu.py` 的 `DEFAULT_MENU`
- 路由注册在 `app/main.py` 顶部 import 并 `include_router`
- 开发记录写在 `MEMORY.md`（时间倒序，最新在最上面）
- 认证: SessionMiddleware + AuthMiddleware，公开路径在 `PUBLIC_PATHS`
- Vite 代理: `/api`, `/login`, `/logout`, `/agent`, `/ai` → `http://127.0.0.1:8000`
- **双数据库模式**: `db/aiops.db` (demo 展示数据) + `db/aiops_real.db` (真实测试数据)，通过顶栏按钮或 `/api/system/db-switch` 切换

## 自动重启规则

- 每次调整后端代码（app/ 下 .py 文件）后，如需重启项目，**自动后台重启**，不要让爸爸手动操作
- 重启方式：用 execute_code 以 Windows 原生 subprocess 后台启动 uvicorn（端口 8000），不要使用 terminal（WSL 未安装）
- 每次调整前端代码（frontend/src/ 下文件）后，需执行 `npm run build` 重新构建，然后重启后端
- 重启前先 kill 旧的 uvicorn 进程，避免端口占用

## 注意事项

- 不要修改 `sxdevops-其他人的项目，用来借鉴的/` 目录，这是参考项目
- `db/aiops.db` 是 SQLite 数据库文件，不要用文本方式读取
- `frontend/node_modules/` 不要提交到 git
- `frontend/dist/` 是构建产物，由 `npm run build` 生成
- `MEMORY.md` 记录开发决策，按时间倒序维护
- **禁止 `taskkill /F /IM pythonw.exe`**：Hermes Agent 自身运行在 `pythonw.exe` 上，执行此命令会杀死 Hermes 导致会话卡死。杀后端进程时必须用 `netstat -ano` 找到端口 8000 对应的 PID，再 `taskkill /F /PID <pid>` 精确杀进程
- **AuthMiddleware 公开路径**：无需 session 的 API 端点（如 OTLP 程序间调用）必须加入 `app/main.py` 的 `PUBLIC_PATHS` 集合，否则会被 303 重定向到 /login
- `terminal` 和 `write_file` 走 WSL bash 但 WSL 未安装，文件读写用 `execute_code`（Python 原生 I/O），`read_file` 对 Windows 反斜杠路径可能误报 not found

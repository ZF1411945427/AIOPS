# AIOps 开发记录

## 2026-06-22: 第二批 5 个新功能实现

### 新增模块

#### 16. HPA 管理
- 通过 K8s API 列出 HorizontalPodAutoscaler（最小/最大/当前/期望副本、CPU 目标/当前利用率）
- 支持按集群和命名空间过滤
- 路径：`/k8s/hpas`

#### 17. 多集群状态概览
- 展示所有 K8s 数据源集群的健康状态：节点数/健康节点/Pod 数/Deployment 数
- 每个集群卡片提供快捷链接到 ConfigMap/Secret/Service/Ingress/HPA
- 路径：`/k8s/overview`

#### 18. STL 周期性异常检测
- `AnomalyConfig` 新增 `period` 字段，默认 12
- 新增 `stl` 算法：移动平均趋势提取 → 周期分片平均 → 残差 3σ 检测
- 异常检测配置表单支持选择 STL 算法并设置周期数
- 适合 CPU/内存等有明确周期特征的指标

#### 19. 数据接入 API
- `POST /api/v1/metrics`：外部系统推送指标数据（支持批量、关联资产、标签）
- `POST /api/v1/events`：外部系统推送事件（支持集群/命名空间/严重级别）
- `GET /api/v1/query/metrics`：查询指标数据（按名称/资产/时间范围过滤）
- 使用 Bearer Token 鉴权，支持 read/write/admin 三级权限
- `/api/v1/docs` API 文档页面
- 侧边栏「系统管理 → API 文档」

#### 20. 指标关联分析
- 计算所有指标两两之间的 Pearson 相关系数
- 热力图矩阵展示（蓝色正相关、红色负相关、无色弱相关）
- Top 20 最强相关排序（强/中/弱/极弱）
- 支持选择时间范围（1h/6h/24h/7d）
- 路径：`/correlation`
- 侧边栏「智能分析 → 指标关联」

### 文件变更
- 新增: `app/routers/api_v1.py`、`app/routers/correlation.py`
- 新增模板: `k8s_hpas.html`、`k8s_overview.html`、`api_docs.html`、`correlation.html`
- 修改: `models.py`（+period）、`anomaly_service.py`（+STL检测）、`anomaly.py`（+算法+周期）、`k8s_resources.py`（+HPA+概览）、`main.py`（+路由）、`base.html`（+导航）、`anomaly.html`（+STL说明+周期字段）

## 2026-06-22: 5 个新功能实现

### 新增模块

#### 11. K8s 集群事件采集
- `K8sEvent` 模型：集群/命名空间/资源名/类型/原因/消息/严重级别
- K8s 采集时自动同步 Events，Warning 标为 warning，Failed/Killing/OOM 标为 critical
- 事件列表页：按集群/类型/级别过滤 + 搜索，分页显示
- 事件统计页：总事件数/警告/严重 + 按类型和原因聚合
- 路径：`/events`、`/events/stats`

#### 12. 钉钉/企微通知渠道
- 通知渠道新增 `dingtalk`（钉钉机器人）和 `wecom`（企微机器人）类型
- 通过 Webhook URL 发送告警通知
- 通知配置表单支持钉钉/企微的 Webhook URL 输入

#### 13. 告警升级完善
- `AlertEscalation` 模型记录每次升级（从/到/原因/时间）
- `escalation_minutes` 从 `SystemConfig` 读取，可在系统设置页面配置
- 告警详情页展示升级历史记录

#### 14. ConfigMap/Secret 管理
- 选择集群 → 列出 ConfigMap / Secret（支持按命名空间过滤）
- ConfigMap 支持查看详情（键值对内容）

- 路径：`/k8s/configmaps`、`/k8s/secrets`

#### 15. Service/Ingress 管理
- 选择集群 → 列出 Service（类型/ClusterIP/端口）
- 列出 Ingress（路由规则/TLS 配置）
- 路径：`/k8s/services`、`/k8s/ingresses`

## 2026-06-18: 项目启动 & MVP 实现

### 已完成模块

#### 1. 项目骨架
- Python + FastAPI 后端
- SQLite 数据库（SQLAlchemy ORM）
- Jinja2 服务端渲染前端
- 深色主题 UI，布局：左侧导航 + 右侧内容区

#### 2. 资产管理（轻量 CMDB）
- 资产类型：host / service / database / middleware
- 字段：名称、类型、IP、状态（online/offline/warning）、标签
- 功能：列表、搜索、过滤、新增、编辑、删除
- 路径：`/assets`

#### 3. 指标监控
- 5 个模拟指标：cpu_usage / memory_usage / disk_usage / network_in / network_out
- 后台线程每 10 秒自动生成模拟数据
- Chart.js 实时折线图（10 秒自动刷新）
- 按资产、指标名称过滤
- 手动"模拟数据"按钮
- 路径：`/metrics`

#### 4. 告警中心
- 告警规则：名称 + 指标 + 条件(gt/lt) + 阈值 + 级别
- 规则管理：新增、启用/禁用、删除
- 告警引擎：后台线程自动检测，超过阈值自动生成告警（相同规则+资产未解决不重复生成）
- 告警状态流转：triggered → acknowledged → resolved
- 手动"触发检查"按钮
- 路径：`/alerts`（告警列表）、`/alerts/rules`（规则管理）

#### 5. 总览看板
- 4 个统计卡片：资产总数、在线数量、未处理告警、告警规则数
- 近期告警列表
- 资产类型分布
- 路径：`/`

### 技术栈
- 后端：FastAPI + SQLite + SQLAlchemy
- 前端：Jinja2 + 原生 CSS + Chart.js
- 运行方式：`python run.py`，默认 `http://localhost:8000`

## 2026-06-18: Phase 1 完善 - 认证/通知/用户管理/告警静默

### 新增模块

#### 6. 用户认证
- User 模型（用户名、密码哈希、角色）
- 登录/登出页面
- SessionMiddleware + AuthMiddleware 鉴权
- 默认管理员：admin / admin123
- 路径：`/login`

#### 7. 用户管理
- admin 可添加/删除用户
- 三种角色：管理员 / 运维人员 / 只读用户
- 路径：`/users`

#### 8. 通知渠道
- 三种类型：邮件(SMTP)、Webhook、日志
- 启动时自动创建默认日志渠道
- 告警触发时自动发送通知
- 通知记录与告警关联
- 路径：`/notifications`

#### 9. 告警静默
- 规则级别静默（默认 30 分钟）
- 静默期内跳过该规则的检查
- 规则列表显示静默状态，可取消
- 路径：`/alerts/rules`

#### 10. 告警详情
- 点击告警 ID 进入详情页
- 展示完整告警信息、关联资产、通知记录
- 路径：`/alerts/{id}`

### 技术优化
- `run.py` 恢复 `reload=True`，改代码热加载
- 启动命令改用 `Start-Process -WindowStyle Hidden`，无阻塞无输出
- 修复路由顺序冲突：`/{alert_id}` 与 `/rules` 的匹配问题

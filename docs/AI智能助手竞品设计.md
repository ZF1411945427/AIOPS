2. 对话双模式切换（底部 Chat/Agent）
Chat 普通对话模式：纯问答聊天，无自动化任务拆解。
Agent 运维自动化模式（当前截图使用）：
用户输入运维问题（如 “查 CPU 高占用进程”）。
Agent 自动拆解分步运维任务，展示任务进度、完成状态、紧急等级。
任务步骤带勾选完成标记，支持展开 / 收起单步执行日志。
自动生成 Linux 运维命令，远程在选定设备执行。
自动解析命令输出结果，结构化提炼异常、Top 进程、风险结论。
3. 远程设备管理与执行
多设备绑定：添加多台 Linux 主机（ssh 通道，截图示例 docker-linux-sshd 容器）。
会话绑定设备，执行命令自动下发至选中设备。
设备标签可视化展示，支持删除设备。
实时执行 Linux 系统命令（top、ps、各类交换机 / 服务器运维指令），回显原始终端输出。
4. Agent 自动化任务核心能力（截图核心特色）
任务拆解引擎：1 个运维问题自动拆分为多步骤流水线，展示进度百分比、总步数。
步骤折叠 / 展开：单步日志可收起、展开，区分「摘要结论」和「原始终端输出」。
自动数据分析：对 ps/top 等系统命令输出做结构化解析，提取高占用进程、PID、CPU 使用率、异常点。
异常自动识别：对比进程资源占用，自动标记高负载异常进程并输出结论。
任务中断控制：底部红色停止按钮，随时终止正在执行的自动化运维流程。
5. UI 交互细节
用户提问：蓝色右浮气泡；AI Agent 输出：白色卡片结构化任务面板。
进度状态可视化：绿色√标识已完成步骤，进度条文本3/3 100%。

快捷键支持：Ctrl+Enter 发送消息，Enter 换行，@唤起设备选择器。
大模型切换：下拉选择不同 LLM 模型（qwen2.5:7b 等）。
三、页面数据流流程（开发参考）
用户在输入框输入运维问题 → 选择目标 Linux 设备 → 切换 Agent 模式发送。
后端 LLM Agent 接收需求，拆解多步运维任务流水线。
Agent 按步骤生成对应 shell 运维命令，通过 ssh 通道下发到目标设备执行。
捕获终端原始输出日志，返回前端渲染。
LLM 二次解析原始日志，提炼结构化摘要、异常结论，分层展示（摘要在上，原始日志可折叠查看）。
全部步骤执行完成后，任务进度标记 100%，展示完整运维分析报告。
整条问答会话持久存入左侧会话列表，支持后续检索、复现。

三、业务数据流完整流程（开发逻辑参考）
流程 1：新建运维排查会话（截图示例场景：查询高 CPU 进程）
用户点击左侧+新建，生成空白会话
底部点击添加设备，录入 127.0.0.1 SSH 信息，绑定设备
底部 Tab 切换至Agent模式
输入框输入：cpu利用率过高是哪个进程引起的，Ctrl+Enter 发送
后端接收需求，调用运维 Agent LLM 拆解任务：拆分为 1 个主步骤 + 1 个子排查步骤
前端渲染运维任务进度卡片，显示紧急、3/3、0% 进度
Agent 自动生成命令ps aux --sort=-%cpu | head -15，下发 SSH 设备执行
流式返回原始 ps 终端日志，同步推送至前端折叠面板
LLM 二次分析日志，提取高占用 PID、进程、CPU 占比，生成异常结论
前端渲染步骤摘要、异常识别文本，步骤标记√完成，进度更新至 100%
整条对话自动存入当前会话，左侧会话列表实时更新，下次打开可复现完整排查记录
流程 2：中断任务
Agent 自动化任务执行中，用户点击底部红色停止按钮
后端立即终止 LLM 调用、关闭 SSH 命令通道
前端停止流式渲染，卡片标记任务「已手动终止」，展示当前已完成步骤
流程 3：Chat 普通聊天
Tab 切换Chat，无需绑定设备即可发送消息
仅调用基础对话 LLM，不生成命令、不执行 SSH、不渲染任务进度卡片

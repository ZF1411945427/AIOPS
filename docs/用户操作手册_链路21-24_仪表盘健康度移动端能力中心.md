# AIOps 用户操作手册 — 链路21-24：仪表盘 & 资产健康度 & 移动端闭环 & Agent能力中心

> **适用角色**：运维工程师、SRE、系统管理员
> **前置条件**：已登录系统，拥有相关菜单权限
> **预计耗时**：约 15-20 分钟

---

## 概述

本手册覆盖四条链路：
- **链路21**：仪表盘拖拽自定义
- **链路22**：资产健康度评分与变更跟踪
- **链路23**：移动端运维闭环
- **链路24**：Agent 能力中心

> **链路21 仪表盘**　①查看预置 → ②创建自定义 → ③拖拽卡片 → ④设默认
> **链路22 资产健康**　①查看评分 → ②批量扫描 → ③变更历史
> **链路23 移动端**　①日志搜索 → ②告警批量 → ③创建故障单 → ④AI会话
> **链路24 Agent能力**　①查看清单 → ②探索工具 → ③风险分级
>

# 链路21：仪表盘拖拽自定义

## 第一步：查看预置仪表盘

### 操作页面
**侧边栏** → **值班驾驶舱** → **运行概览** → **自定义仪表盘**

### 操作步骤

1. **查看仪表盘列表**
   - 页面显示所有仪表盘布局
   - 每个布局显示：
     - **布局名称**
     - **描述**
     - **是否默认**
     - **创建时间**

2. **查看预置模板**
   - 系统提供 3 套预置模板：
     - **运维总览**：资产概览 + 告警趋势 + 故障统计
     - **告警中心**：告警概览 + 告警级别 + 最新告警
     - **SRE 看板**：SLO 状态 + 自愈效果 + MTTR 趋势

3. **切换布局**
   - 点击布局名称，切换到对应仪表盘
   - 页面显示该布局的所有卡片

### 看到什么
- 仪表盘列表显示所有布局
- 预置模板以卡片形式展示
- 切换布局后页面显示对应的卡片网格

## 第三步：拖拽添加卡片

### 操作页面
**侧边栏** → **值班驾驶舱** → **运行概览** → **自定义仪表盘**

### 操作步骤

1. **打开卡片库**
   - 点击页面右上角「添加卡片」按钮
   - 左侧弹出卡片库面板，显示 16 种卡片类型：

| 序号 | 类型 | 标题 |
|------|------|------|
| 1 | stat-asset | 资产概览 |
| 2 | stat-alert | 告警概览 |
| 3 | stat-incident | 故障概览 |
| 4 | stat-health | 健康评分 |
| 5 | chart-alert-trend | 告警趋势 |
| 6 | chart-asset-type | 资产分布 |
| 7 | chart-severity | 告警级别 |
| 8 | chart-health-trend | 健康趋势 |
| 9 | list-recent-alerts | 最新告警 |
| 10 | list-recent-incidents | 最新故障 |
| 11 | chart-mttr | MTTR 趋势 |
| 12 | chart-remediation | 自愈统计 |
| 13 | stat-datasource | 数据源 |
| 14 | stat-rule | 告警规则 |
| 15 | chart-ai-tool | AI 工具调用 |
| 16 | chart-notification | 通知统计 |

2. **拖拽卡片到画布**
   - 从卡片库拖拽卡片到右侧画布（gridstack 网格）
   - 卡片自动吸附到网格位置

3. **调整卡片位置和大小**
   - 拖拽卡片边缘调整大小
   - 拖拽卡片标题栏调整位置

4. **配置卡片数据源**
   - 点击卡片右上角「设置」按钮
   - 弹窗配置数据源：
     - **数据源**：选择 API 端点
     - **刷新间隔**：选择自动刷新间隔（5分钟 / 15分钟 / 30分钟 / 手动）

5. **保存布局**
   - 点击页面右上角「保存」按钮
   - 系统调用 `POST /dashboard-config/layouts` 保存布局配置

### 看到什么
- 画布显示拖拽的卡片网格
- 卡片自动加载数据源
- 保存后布局持久化

# 链路22：资产健康度评分与变更跟踪

## 第一步：查看资产健康评分

### 操作页面
**侧边栏** → **运维工作台** → **资产管理** → **资产列表**

### 操作步骤

1. **查看资产列表**
   - 页面显示所有资产
   - 每条资产显示：
     - **资产名称**
     - **IP 地址**
     - **CI 类型**
     - **健康评分徽章**：绿色（≥80）/ 黄色（60-79）/ 红色（<60）

2. **查看健康评分详情**
   - 点击某条资产
   - 右侧弹出详情抽屉，显示：
     - **健康评分**：0-100 分
     - **评分状态**：healthy / warning / danger
     - **评分详情**：
       - 基础分：40 分（在线）/ 0 分（离线）
       - CPU 分：最高 20 分
       - 内存分：最高 20 分
       - 磁盘分：最高 20 分
       - 告警惩罚：-10 分/条（triggered）/ -2 分/条（acknowledged）

### 看到什么
- 资产列表显示健康评分徽章
- 详情页显示评分计算详情

---
**下一步：触发健康扫描** →

## 第二步：触发健康扫描

### 操作页面
**侧边栏** → **运维工作台** → **资产管理** → **资产列表**

### 操作步骤

1. **点击「健康扫描」按钮**
   - 在资产列表页面，点击「健康扫描」按钮
   - 系统调用 `POST /assets/api/health-scan`

2. **自动执行扫描**
   - 后台执行 `scan_asset_health_changes()`：
     - 批量遍历 online 资产
     - 计算每个资产的健康评分
     - 对比上次状态变化时的评分
     - 如果状态变化（如 green → yellow），写入 `AssetChangeLog`

3. **查看扫描结果**
   - 页面刷新，显示最新的健康评分
   - 变化的资产评分徽章更新

### 看到什么
- Toast 提示「健康扫描完成」
- 资产列表的健康评分徽章更新
- 状态变化的资产显示变更标记

## 第三步：查看变更历史

### 操作页面
**侧边栏** → **运维工作台** → **资产管理** → **资产详情** → **变更历史**

### 操作步骤

1. **查看变更记录**
   - 在资产详情页，点击「变更历史」Tab
   - 页面显示该资产的所有变更记录（`AssetChangeLog`）
   - 每条记录显示：
     - **变更类型**：`field_update`（字段更新）/ `health_status`（健康状态变化）
     - **变更字段**：修改的字段名
     - **旧值 → 新值**
     - **操作人**：`user_id`（自动记录）
     - **变更时间**：`created_at`

2. **筛选变更记录**
   - **类型筛选**：全部 / 字段更新 / 健康状态变化
   - **时间范围**：选择时间范围

### 看到什么
- 变更历史列表按时间倒序排列
- 每条变更记录完整追踪字段变化
- 健康状态变化以颜色标记（green → yellow → red）

### 结果
资产健康度闭环：评分计算 → 健康扫描 → 状态变化 → 变更记录 → 审计追踪。

# 链路23：移动端运维闭环

## 第一步：移动端日志搜索

### 操作页面
**移动端** → **运维工具** → **日志搜索**

### 操作步骤

1. **打开日志搜索页**
   - 在移动端底部导航栏，点击「工具」Tab
   - 点击「日志搜索」卡片

2. **输入搜索条件**
   - **关键词**：输入搜索关键词（如 `error`）
   - **级别**：选择日志级别（ERROR / WARN / INFO / DEBUG）
   - **数据源**：选择数据源（如 `syslog` / `nginx`）
   - **时间范围**：选择开始/结束时间

3. **执行搜索**
   - 点击「搜索」按钮
   - 系统调用 `GET /logs/api/search?keyword=&level=&source=&start=&end=&page=`

4. **查看搜索结果**
   - 页面显示日志列表
   - 每条日志显示：
     - **时间戳**
     - **级别**：ERROR（红色）/ WARN（橙色）/ INFO（蓝色）/ DEBUG（灰色）
     - **内容**：日志消息

5. **查看日志详情**
   - 点击某条日志
   - 弹窗显示完整日志内容
   - 支持 JSON 格式化显示
   - 支持复制到剪贴板

### 看到什么
- 日志列表按时间倒序排列
- 级别以颜色标记
- 详情页支持 JSON 格式化和复制

---
**下一步：移动端告警批量操作** →

## 第二步：移动端告警批量操作

### 操作页面
**移动端** → **告警中心** → **告警列表**

### 操作步骤

1. **进入多选模式**
   - 在告警列表页，长按某条告警
   - 进入多选模式（checkbox 显示）

2. **选择告警**
   - 勾选多条告警
   - 底部工具栏显示「已选 X 条」

3. **批量确认**
   - 点击「批量确认」按钮
   - 系统调用 `POST /alerts/api/batch-acknowledge`
   - 所有选中告警状态变为 `acknowledged`

4. **批量解决**
   - 点击「批量解决」按钮
   - 弹出二次确认对话框
   - 点击「确认」
   - 系统调用 `POST /alerts/api/batch-resolve`
   - 所有选中告警状态变为 `resolved`

### 看到什么
- 多选模式下 checkbox 选中告警
- 批量操作成功后 Toast 提示
- 告警状态批量更新

## 第三步：移动端创建故障单

### 操作页面
**移动端** → **故障管理** → **故障列表** → **新建故障单**

### 操作步骤

1. **进入故障列表**
   - 在移动端底部导航栏，点击「故障」Tab
   - 显示故障列表

2. **点击「新建故障单」按钮**
   - 页面右上角的「+」按钮

3. **填写故障信息**
   - **标题**：输入故障标题（如 "API 响应超时"）
   - **严重级别**：选择（critical / high / medium / low）
   - **影响范围**：选择受影响的资产
   - **紧急联系人**：输入联系人姓名和电话
   - **描述**：详细描述故障现象

4. **提交故障单**
   - 点击「提交」按钮
   - 系统调用 `POST /incidents/api/create`
   - 成功后 Toast 提示并返回故障列表

### 看到什么
- 表单页面显示故障信息输入框
- 提交成功后 Toast 提示「创建成功」
- 故障列表显示新创建的故障单

## 第四步：移动端 AI 会话管理

### 操作页面
**移动端** → **AI 助手** → **会话列表**

### 操作步骤

1. **查看历史会话**
   - 在 AI 助手页面，点击右上角「历史」按钮
   - 页面显示历史会话列表

- 点击会话跳转到聊天页面

## 第三步：理解工具分类

## 常见问题

### Q1：仪表盘卡片数据为空？
- **检查**：卡片数据源 API 是否正常返回
- **检查**：是否有对应的告警/故障/资产数据
- **检查**：网络是否连通

### Q2：资产健康评分不准确？
- **检查**：资产是否在线（`status=online`）
- **检查**：是否有活跃告警（`Alert.status=triggered/acknowledged`）
- **检查**：指标数据是否最新（CPU/MEM/DISK 最新值）

### Q3：移动端日志搜索无结果？
- **检查**：数据源是否配置正确
- **检查**：关键词是否正确
- **检查**：时间范围是否正确

### Q4：Agent 能力中心工具数量不对？
- **检查**：`mcp_tools.py` 中 `@register_mcp_tool` 装饰器数量
- **检查**：`/agent/api/capabilities` API 是否正常返回

```

事件：电商平台"秒杀下单"高峰，支付服务开始出现大量超时

- 01:30:17 — 第 1 条告警：payment-gateway-01 CPU 92%

- 01:34:20 — 用户支付页面开始出现大面积超时

  client → CDN → api-gateway → auth-service → payment-gateway

故障摘要（50 字）：
支付服务因灰度发布配置变更引发 DB 连接池压力，
叠加 redis 内存减少导致缓存击穿，演变为雪崩式故障，

- Why3：为什么 DB 之前能承载 100 连接，现在不能？

- Why4：为什么 redis 内存被减少？

## 场景复杂度总结

| 维度 | 本场景特点 |
|------|-----------|
| **告警数量** | 7 个以上关键告警，集中在 5 分钟内爆发 |

| **知识闭环** | 故障 → 根因 → 改进项 → 验证任务 → SOP 更新 |

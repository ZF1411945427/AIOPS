### M2. HTTPS 强制（P0 发布前必做）

- **现状**：`mobile/src/api/config.js` 默认 `http://127.0.0.1:8000`（开发环境）
- **生产**：登录页服务器地址输入框必须填 `https://<生产域名>`
- **后端配合**：FastAPI 生产环境需配 SSL 证书（Nginx 反代或 uvicorn `--ssl-keyfile`）
- **验证**：抓包工具（Charles/Fiddler）应看不到明文 token

### M3. Token 存储安全（P1 建议做）

- **现状**：`auth_token` / `biometric_token` / `session_cookie` 明文存 `uni.storage`
- **风险**：越狱/Root 设备可被提取，跨应用读取
- **方案**：
  - H5 端：localStorage 本就明文，只能靠 HTTPS + 短时效 token 缓解
  - App 端：用 `uni.setStorageSync` 前调 native crypto 加密（Android Keystore / iOS Keychain）
  - 小程序端：微信 storage 本就沙箱隔离，无需额外加密
- **成本**：跨端兼容性麻烦，建议仅 App 端做，H5/小程序暂缓

### M4. WebSocket Token 传输（P1 建议做）

- **现状**：`mobile/src/utils/ws.js` 用 `?token=xxx` 拼 URL，会被 Nginx access log 记录
- **方案**：改用 `Sec-WebSocket-Protocol` 子协议头传 token
  ```js
  // ws.js 改造
  const ws = new WebSocket(url, ['bearer', token])  // 走子协议头
  // 后端读取 Sec-WebSocket-Protocol 转发到 query
  ```
- **成本**：后端 ws 服务需配合解析子协议头，约 0.5 人天

### M5. 代码混淆与反逆向（P2 可暂缓）

- **H5 端**：uni-app build 后已压缩 + 变量名混淆，无需额外投入
- **App 端**：DCloud 加密打包（5+ 基座）已含基础保护
- **小程序端**：微信自动混淆，无需处理
- **高安全场景**：App 端可加 DCloud 「安全加固」服务（需付费）

### M6. 权限最小化（P0 发布前必做）

`manifest.json` 当前声明的 Android 权限：
| 权限 | 用途 | 是否必要 |
|------|------|---------|
| INTERNET | 网络请求 | ✅ 必要 |
| CAMERA | 资产扫码 | ✅ 必要 |
| ACCESS_FINE_LOCATION | 现场打卡 | ✅ 必要 |
| NFC | 资产扫码（NFC 标签） | ⚠️ 若不用 NFC 可删 |
| VIBRATE | 告警震动 | ✅ 必要 |
| WRITE_EXTERNAL_STORAGE | 日志/导出 | ⚠️ Android 11+ 用 Scoped Storage，可删 |
| READ_EXTERNAL_STORAGE | 同上 | ⚠️ 同上 |
| ACCESS_NETWORK_STATE | 网络状态检测 | ✅ 必要 |
| WAKE_LOCK | 推送保活 | ✅ 必要 |
| USE_FINGERPRINT | 生物识别 | ✅ 必要 |

**建议**：发布前删掉 `WRITE/READ_EXTERNAL_STORAGE`（Android 11+ 已废弃），改用 Scoped Storage API。

### M7. 发布前 SAST 扫描（P0 发布前必做）

- **工具**：`npm audit`（扫 npm 依赖 CVE）+ SonarQube（扫 JS/Vue 源码）
- **当前已知风险**：移动端用 `md5` 算设备指纹（`biometric.js:49`），SAST 会报弱哈希
  - **处理**：加注释 `// nosec: 设备指纹非安全场景，仅用于唯一性标识`，或改用 `sha256`（`crypto.js` 已导出）
- **建议**：打磨期不动，发布前统一改 `sha256` 并跑一次 `npm audit`

## 一、背景

在 AI 盛行的环境下，系统部署到客户环境后面临两类核心风险：

1. **知识产权风险**：客户拿到代码进行二次修改售卖
2. **安全验收风险**：客户对交付物做代码扫描（漏洞 / License / 后门 / 逆向），不通过则拒收或扣款

本系统技术栈：**Python（FastAPI 后端）+ Vue（前端 SPA）+ uni-app（移动端 H5）+ AI 模型/推理服务**。

当前防护状态：**源码裸奔，四类防护（L1~L4）全部缺失**，任何客户拿到机器即可 `cat` 源码。

---

## 二、需求一：知识产权保护（防二次修改售卖）

### 2.1 风险描述

客户拿到部署包后，直接复制源码、修改品牌、二次售卖，造成：
- 直接经济损失（潜在客户被截胡）
- 品牌稀释（劣质仿品影响口碑）
- 核心算法/模型权重外泄（AI 项目核心资产）

### 2.2 防护方案（四层架构，由本到标）

| 层级 | 方案 | 做法 | 适用场景 | 治本程度 |
|------|------|------|---------|---------|
| **L4 架构隔离** | SaaS 化 / 混合部署 | 核心 AI 服务（模型推理、算法引擎）放云端，客户环境只跑调用层 | 客户接受联网 | ★★★★★ 治本 |
| **L3 代码加密** | Cython / Pyarmor | 核心 `.py` 编译成 `.pyd`（C 扩展）或 Pyarmor 字节码加密 + 运行时解密 | 离线交付 | ★★★★ |
| **L2 运行时校验** | License + 硬件指纹 | MAC + CPU序列号 + 主板 SN 生成机器码，License 绑定机器，到期降级 | 所有交付 | ★★★ |
| **L1 反调试** | 检测调试器/VM/抓包 | 增加逆向成本，配合 L3 使用 | 高安全场景 | ★★ |

### 2.3 Python 后端具体手段

- **Cython 编译**：核心算法包编译为 `.pyd`，反编译几乎不可能
- **Pyarmor 加密**：字节码加密 + 运行时解密，内置授权机制
- **PyInstaller / Nuitka 打包**：打成单一 exe，源码不裸露
- **禁止行为**：只发 `.py` 源码 = 等于送代码

### 2.4 Vue 前端

- `npm run build` 后 JS 本就是混淆压缩的，够用
- 关键业务逻辑不放前端，统一调后端接口

### 2.5 法律层（兜底必备）

- 合同明确**著作权归属、禁止反编译、禁止二次分发、违约赔偿**
- 交付物标注水印/版权声明
- 保留源码哈希作为权属证据

### 2.6 推荐组合（本项目）

```
核心 AI 服务云端化（L4） + Python Cython/Pyarmor 编译（L3） + License 硬件绑定（L2） + 合同条款（法律）
```

**最低成本方案**：Pyarmor 混淆 + 硬件指纹 License + 合同，1-2 天可落地。

## 三、需求二：代码扫描防护

### 3.1 四类扫描防护矩阵

| 扫描类型 | 风险点 | 防护手段 | 实施成本 | 优先级 |
|---------|--------|---------|---------|--------|
| **SAST/DAST 漏洞扫描** | 扫出 SQL 注入、XSS 等漏洞 → 客户拒收/扣款 | 开发期自查 + 安全基线 | 中 | P0 |
| **License 合规扫描** | 扫到 GPL/AGPL → 客户要求开源全部代码（copyleft 传染） | 组件清单管控 + 替换 | 低 | P0 |
| **恶意代码/后门扫描** | 误报后门 → 交付卡关（国企/金融常见） | 杀软白名单 + 数字签名 | 低 | P1 |
| **逆向工程扫描** | 核心算法被扒 → 被二次售卖 | 代码加密 + 架构隔离 | 高 | P2 |

### 3.2 漏洞扫描防护（SAST/DAST）— P0 最重要

#### 3.2.1 交付前自查命令

```bash
# Python 后端 SAST
bandit -r app/ -f json -o bandit-report.json    # 找注入/弱密码等
pip-audit                                        # 依赖 CVE 扫描
safety check                                     # 同上，互补

# 前端 SAST
npm audit --prefix frontend
npx eslint --ext .js,.vue frontend/src --max-warnings 0

# DAST（启动后端后）
nuclei -u http://localhost:8000 -t cves/
zap-cli quick-scan -s http://localhost:8000
```

#### 3.2.2 常见会被扫出的坑（本项目预测）

- ❌ `debug=True` 没关 → FastAPI/Flask 泄露堆栈
- ❌ SQL 拼接而非参数化 → 注入
- ❌ 默认密钥/弱口令 → 配置漏洞
- ❌ CORS `*` 全开 → 跨域
- ❌ FastAPI `/docs`、`/redoc` 没关 → 接口裸露
- ❌ 依赖包 CVE（尤其老版本 `pydantic`、`fastapi`、`requests`）

**最低标准**：交付前跑完上述工具，报告 0 高危 / 0 中危。

### 3.3 License 合规扫描防护（SCA）— P0 易被忽视

#### 3.3.1 核心风险

用了 GPL/AGPL 组件 → Black Duck 扫出来 → 客户法务要求**开源全部代码**（copyleft 传染性）。

#### 3.3.2 自查命令

```bash
# Python 依赖清单
pip freeze > requirements.txt
pip-licenses --format=csv --output-file=licenses.csv

# 查危险 License
pip-licenses | grep -iE "GPL|AGPL|LGPL|CC-BY-NC"

# 前端
npx license-checker --production --json > licenses.json
# 重点关注 GPL*、AGPL*、CC-BY-NC*
```

#### 3.3.3 Python 高危组件黑名单（出现就得换）

- `psycopg2`（LGPL）→ 改用 `psycopg2-binary` 或 `asyncpg`
- 任何 `AGPL-3.0` 的包
- 某些 `pytorch` / `transformers` 衍生组件

#### 3.3.4 交付物

附 `THIRD_PARTY_LICENSES.txt` + `LICENSE_AUDIT.xlsx`，主动给客户，省得他们扫。**SBOM（软件物料清单）已成为企业级交付标配**。

### 3.4 恶意代码/后门扫描防护 — P1

#### 3.4.1 交付前自证清白

1. **数字签名**：用公司代码签名证书给 `exe`/`whl` 签名
   ```powershell
   signtool sign /f cert.pfx /p xxx /t http://timestamp.digicert.com yourapp.exe
   ```
2. **VirusTotal 预扫描**：把交付包传 https://www.virustotal.com/ 预跑，留存报告
3. **构建可追溯**：CI/CD 构建日志 + commit hash + 构建时间，证明来源
4. **依赖锁版本**：`requirements.lock` + `package-lock.json`，避免供应链投毒
5. **杀软白名单**：提前列进程名、路径、端口，让客户安全部门加白

#### 3.4.2 避坑

Pyarmor 加密后的 `.py` 在某些杀软会误报，提前送测。

### 3.5 逆向工程防护 — P2（与需求一 L1~L4 重合）

## 四、专业名词与话术

### 4.1 专业名词

| 名词 | 释义 |
|------|------|
| **代码混淆** | Code Obfuscation，通过重命名/控制流扁平化等手段让代码难以阅读 |
| **数字版权管理** | DRM (Digital Rights Management) |
| **硬件指纹绑定** | Hardware Fingerprinting / Machine ID Binding |
| **反逆向工程** | Anti-Reverse Engineering |
| **License 授权** | Software Licensing |
| **SaaS 化** | Software as a Service |
| **字节码加密** | Bytecode Encryption |
| **白盒加密** | White-Box Cryptography |
| **SAST** | Static Application Security Testing，静态安全扫描（扫源码） |
| **DAST** | Dynamic Application Security Testing，动态安全扫描（扫运行态） |
| **SCA** | Software Composition Analysis，开源组件合规分析 |
| **SBOM** | Software Bill of Materials，软件物料清单（交付标配） |
| **Copyleft 传染** | GPL 类协议要求衍生作品必须开源 |
| **代码签名** | Code Signing |
